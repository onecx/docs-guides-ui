== Usage of Module Federation in OneCX

In OneCX, *Module federation* is used to load microfrontends, expose reusable components and for dependancy sharing.

Following are important concepts where module federation is involved:

.Features
[.stripes-even,cols="20,~"]
|===
| Name  | Description
| xref:./mf-preloaders.adoc[Preloaders] | Preloaders register Module Federation share scopes
| xref:./mf-shell-router.adoc[Shell router] | Shell router - dynamic content loading
| xref:./mf-dependancy-sharing.adoc[Dependancy sharing] | Build time and runtime negotiation of shared packages
| xref:./mf-slot-components.adoc[Slot components] | Load remote modules or components in slots
| xref:./mf-authentication-service.adoc[Authentication service] | To implementation of custom authentication service
|===
