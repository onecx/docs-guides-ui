== Slot components and Module Federation

In OneCX, *Module federation* is used in *slot components* to load dynamic content either as module or as a script.

While loading content dynamically, slot has -

* *Slot component* - host for dynamic content
* *Slot service* - configures and loads dynamic content in slot component using module federation

It also uses *Remote Component Topic* to communicate between host and remote component.

=== Execution Flow
* *SlotComponents* check *Remote Component Topic* by using *SlotService*
* *SlotService* extract remote component information from *Remote Component Topic*
* *SlotService* finds matching slot definition and extracts component names
* *SlotService* takes decision based on following parameters -

* Exposed module is take from *Remote Component Topic*
* When technology is `WebComponent`, `WebComponentScript`, or `WebComponentModule`, then link `WebcomponentLoaderModule` in the `loadChildren` callback.

.Technology 
[.stripes-even,cols="20,~"]
|===
| Technology | Configuration
| WebComponentModule | type = *module*
| WebComponentScript | type = *script* and set *remoteName* from *Workspace configuration*
| WebComponent | type = *script* and set *remoteName* from *Workspace configuration*
|===

* Based on component Type slotService create component-
    * If *componentType* is present the create *Angular component*
    * Component creation is done by following criteria

.Component creation criteria 
[.stripes-even,cols="20,~"]
|===
| Created Component | Configuration
| Angular Components | when *componentType* is present in configuration, Create angular component, adds styles, production information and ocxInitRemoteComponent lifecycle hook 
| WebComponent | When technology is `WebComponentModule` or `WebComponent` then create webcomponent with styles, and adds production information
| WebComponentScript | Loads script via *module federation* and creates custom element and appends to DOM
|===
