== Slots and Module Federation

*Slots* are a mechanism in OneCX to dynamically render *remote components* in *designated locations* on a page using *Module Federation*. *Slots* are configurable at *runtime*.

=== Slot rendering process

Example flow:

* *App1* owns the page and defines the *slot location*
* *App2* exposes *remote components* (via module federation as remote module) that can be placed into that slot
* Both *App1* and *App2* applications are used workspace
* *App1* slot is *activated* in the workspace
* *App2* remote component is assigned to *App1* slot
* When *App1*'s slot is displayed, it will dynamically load the *App2*'s component using module federation and render it in the slot location using appropriate technology defined in remote component configuration

=== Slot rendering based on technology
[.stripes-even,cols="20,~"]
|===
| Technology | Description

| Angular
a|
*Steps:*

* Create component dynamically
* Initialize component with metadata and permissions
* Add component to the DOM

| WebComponents
a|
*Steps:*

* Creates a native HTML element
* Sets metadata and permissions
* Add the HTML element to the DOM
