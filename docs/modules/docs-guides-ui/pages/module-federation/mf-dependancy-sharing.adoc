[#mf-dependancy-sharing]
= Module Federation dependancy sharing in OneCX

In OneCX, Dependency sharing is done using *Module federation* at two times:

 * **Build Time**
 * **Runtime**

== Build Time configuration
This phase defines which packages are shared via webpack configuration

=== In webpack.config.js of shell-ui
* Configuration is defined `webpack.config.js` of *shell-ui*
* In shared configuration, following packages are shared:
    - Angular packages, 
    - OneCX common packages and 
    - rxjs and primeNG packages are shared

.Example
[%collapsible]
====
[source,javascript]
----
const { share, withModuleFederationPlugin } = require('@angular-architects/module-federation/webpack')

const webpackConfig = {
  ...withModuleFederationPlugin({
    name: 'onecx-shell-ui',
    exposes: { ... },
    
    shared: share({      
      '@angular/core': { requiredVersion: 'auto', includeSecondaries: true },
      '@angular/common': {
        requiredVersion: 'auto',
        includeSecondaries: { skip: ['@angular/common/http/testing'] }
      },
      '@angular/common/http': { requiredVersion: 'auto', includeSecondaries: true },
      '@angular/elements': { requiredVersion: 'auto', includeSecondaries: true },
      '@angular/forms': { requiredVersion: 'auto', includeSecondaries: true },
      '@angular/platform-browser': { requiredVersion: 'auto', includeSecondaries: true },
      '@angular/router': { requiredVersion: 'auto', includeSecondaries: true },
      '@angular-architects/module-federation-tools': { requiredVersion: 'auto', includeSecondaries: true },
      '@ngx-translate/core': { requiredVersion: 'auto' },
      '@ngx-translate/http-loader': { requiredVersion: 'auto' },
      'primeng': { requiredVersion: 'auto', includeSecondaries: true },
      'rxjs': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/accelerator': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/angular-accelerator': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/angular-auth': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/angular-integration-interface': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/angular-remote-components': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/angular-utils': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/angular-webcomponents': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/integration-interface': { requiredVersion: 'auto', includeSecondaries: true },
      '@onecx/portal-layout-styles': { requiredVersion: 'auto', includeSecondaries: true }
    })
  })
}
----
**Notes:**

- Each package in `shared: {...}` will be available for sharing
- `requiredVersion: 'auto'` - Use exact version from package.json
- `includeSecondaries: true` - Also share dependencies of these packages
====

=== Runtime negotiation and loading
In this phase, *Shell-ui* and *Microfrontend* negotiate which version of shared package to load and then load the package. This is done by *Module federation* at runtime.

.Execution flow
* In `webpack.config.js` of *preloaders*, `ModuleFederationPlugin` is configured with `shared` configuration. This registers the share scopes and shared packages for *Shell-ui* with *Microfrontend*
* Preloaders share ALL packages
* The preloaders use *magicChar* to ensure they are chosen first

.Example
[source,javascript]
----
// webpack.config.js of preloaders
const { withModuleFederationPlugin, shareAll } = require('@angular-architects/module-federation/webpack')

const webpackConfig = {
  ...withModuleFederationPlugin({
    name: magicChar + 'onecx-angular-20-loader', // Magic char = 0x10ffff (highest unicode)
    filename: 'remoteEntry.js',
    exposes: {
      ['./Angular20Loader']: 'src/main.ts'
    },
    
    shared: shareAll({ // Share all packages from package.json
      requiredVersion: 'auto', 
      includeSecondaries: true 
    }, undefined, './')
  })
}
----

**Notes**
The preloaders use `magicChar + 'onecx-angular-20-loader'` to ensure they're chosen first during negotiation. Unicode codepoint `0x10ffff` is the highest possible character, so string comparison will always select the preloader.

=== How dependancies are negotiated
* *Step 1*: Shell-ui receives shared scope from remoteEntry of microfrontend
* *Step 2*: *Shell-ui* and *Microfrontend* exchange their shared package versions during runtime
* *Step 3*: For each shared package, the highest compatible version is selected based on semver rules

.Example - Shell Shared scope
[source, javascript]
{
'@angular/core': { version: '20.x', shared: true },
'@angular/router': { version: '20.x', shared: true },
'rxjs': { version: '7.x', shared: true },
'custom-lib': '1.x.x',
 // ... its shared packages
 }

.Example - Dashboard shared scope
[source, javascript]
{
'@angular/core': { version: '20.x', shared: true },
'@angular/router': { version: '20.x', shared: true },
'rxjs': { version: '7.x', shared: true },
'custom-lib': '2.x.x'
 // ... its shared packages
 }

.Result of negotiation
.Features
[.stripes-even,cols="20,20,20,20"]
|===
|Library Name  | Shell version  | Dashboard version  | Final version
|@angular/core  | 20.2.0  | 20.1.0  | 20.2.0
|custom-lib  | 1.x.x  | 2.x.x  | 2.x.x
|===

.Example - Result of negotiation
[source, javascript]
{
    '@angular/core': Shell's instance,
    '@angular/router': Shell's instance,
    'rxjs': Shell's instance,
    'custom-lib': Dashboard's own copy
}
